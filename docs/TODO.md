- [ ] RAG 与模型常识的双模式回答策略
  - 目标：解决“选了知识库就只能按上下文回答，不选知识库才可用模型常识”的单一策略问题。
  - 方案：支持 `strict` 与 `hybrid` 两种模式。
  - 设计要点：
    - Node1 增加来源决策字段（建议：`answerSource`，取值如 `RAG_ONLY | MODEL_ONLY | RAG_PREFERRED`）。

    - Node3 校验结果是否符合来源策略（是否越界、是否缺证据）。
    - 前端增加模式开关并透传给执行链路。
